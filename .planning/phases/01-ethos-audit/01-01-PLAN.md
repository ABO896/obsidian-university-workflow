From .planning/codebase/CONVENTIONS.md:

Config-Driven Design Rules:
- All folder names, labels, years, and type identifiers MUST come from universityConfig.js at runtime
- Never hardcode strings that exist in config
- Feature flags (features.parcial) must be checked before parcial-related branches

Templater API Conventions — forbidden vs. required:
  new Date().toISOString()    → tp.date.now("YYYY-MM-DD")
  window.prompt(...)          → await tp.system.prompt(...)
  fetch(...)                  → await tp.web.request(...)
  app.vault.getName()         → tp.file.folder()
  bare app.* in templates     → tp.app.*

Async rule: all tp.* async functions must be awaited. tp.date.now() is synchronous — no await.

Bootstrap pattern (canonical, per CONTEXT.md):
  Creation templates: await tp.user.templateBootstrap(tp, { requireNewFile: true })
  Guard: if (!ctx) return;
  Deviations from this pattern are Templater-native violations.
