<!-- Pre-catalogued expansion opportunities from RESEARCH.md AUDIT-04 section. Executor should use this list in full. -->

E-01 (actionable): Concept Note Template.md missing requireNewFile guard
  — Also documented as a critical Templater-native violation. The expansion framing: adding { requireNewFile: true } is a one-line fix that closes a data-loss risk class. High impact.

E-02 (actionable): Shared reviewIntervals fallback constant
  — Concept Note Template.md and Quick Create Concept.md each hardcode { easy: 14, medium: 7, hard: 3, blank: 1 }. Single access path via config?.schema?.reviewIntervals eliminates duplication and drift.

E-03 (actionable): Update Note Status.md missing "raw" in fallback statuses
  — Also documented as a critical config-purity violation. Expansion framing: fixing the fallback array closes a silent-failure path for notes with status:"raw".

E-04 (actionable, low priority): Redundant fallback on reviewIntervals.medium in date offset
  — Quick Create Concept.md and Concept Note Template.md use tp.date.now("YYYY-MM-DD", reviewIntervals.medium ?? 7). The ?? 7 is redundant if config is always present. Very low impact.

E-05 (out of scope): No template for creating a new Subject (folder + hub in one step)
  — Users currently must run Subject Hub template, select a non-existent subject, and manually type the name. A dedicated New Subject template could create the folder, generate the hub note, and open it in a single action.

E-06 (out of scope): No Study Session or daily-capture template
  — The system has lecture notes, concept notes, and exam prep, but no lightweight "I studied today" journal entry that cross-links to concepts reviewed. Mark Reviewed covers the spaced-repetition side, but there is no narrative capture for study sessions.

E-07 (out of scope): getUniversityContext.js lazy-init state persists for vault lifetime
  — Module-level _initialized flag means config changes (adding a new year) require an Obsidian vault reload. ARCHITECTURE.md documents this as a known constraint. A "Reset context cache" utility template calling a reset() export on getUniversityContext.js would address this for mid-session config edits.

E-08 (actionable): Subject Hub.md "updated" frontmatter key is non-standard
  — The template generates "updated: YYYY-MM-DD" in frontmatter, but "updated" is not in CONVENTIONS.md's required key order. If intentional, add it to conventions; if unintentional, remove it from the template.

E-09 (out of scope): Mobile usage blockers not surfaced in templates
  — tp.user.* scripts are unavailable on Obsidian mobile. No template currently shows a user-facing notice when run on mobile. A mobile feature flag in universityConfig.js plus a bootstrap-level guard could prevent confusing silent failures.

Ordering by apparent impact (for final list ordering):
  1. E-01 (data-loss risk, one-line fix)
  2. E-03 (silent failure for "raw" notes, one-line fix)
  3. E-08 (non-standard frontmatter key — affects every Subject Hub note)
  4. E-02 (config drift between two templates, reduces to single source)
  5. E-05 (workflow gap — new subject creation is cumbersome) [out of scope]
  6. E-06 (workflow gap — no study session capture) [out of scope]
  7. E-07 (config cache lifetime gotcha) [out of scope]
  8. E-09 (mobile silent failure) [out of scope]
  9. E-04 (low-priority redundant fallback)
