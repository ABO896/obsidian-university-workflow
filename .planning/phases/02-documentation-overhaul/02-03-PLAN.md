       - Answer prompts in this order: Year -> `Year 1`, Subject -> `Physics I`, Tema -> `Waves`, Topic -> `Electromagnetic Spectrum`.

    4. `### Stage 2: Mid-lecture — capture a concept on the fly` subsection:
       - Highlight a term in the lecture note (e.g., `Photoelectric Effect`).
       - Run the **Quick Create Concept** template.
       - Confirm the concept name when prompted (the selection pre-fills it).
       - Result: a new concept note is created at `University/Year 1/Physics I/Temas/Waves/Photoelectric Effect.md`, a `[[Photoelectric Effect]]` link appears at the cursor in the lecture note, and the lecture note's `concepts` frontmatter array now contains `Photoelectric Effect`.
       - Note: course/year/tema are inherited from the lecture note's frontmatter, so you are not asked to pick them again.
    5. `### Stage 3: After class — link the remaining concepts` subsection:
       - Open the lecture note.
       - Run the **Link Concepts to Note** template.
       - A multi-select picker appears listing every concept note in Physics I. Already-linked concepts are marked with a `✓` prefix.
       - Select the additional concepts to add.
       - Result: only the lecture note's `concepts` frontmatter is updated — the body of the note is untouched.
    6. `### Stage 4: Before the exam — build a parcial prep note` subsection:
       - Create a new untitled note.
       - Run the **Parcial Prep Note** template.
       - Answer prompts: Year -> `Year 1`, Subject -> `Physics I`. If `features.parcial: true` in your config, also pick the exam period (e.g., `Parcial 2`).
       - Result: a study guide note is created with Dataview tables that auto-populate with every lecture and every concept from Physics I / Year 1. You then tab through three cursor stops: `Topics to Cover`, `Summary Notes`, `Practice Questions`.
       - Briefly note: when `features.parcial: false` (default), the file is placed at the subject root; when `true`, it is placed inside `Parciales/<Parcial>/`.
    7. A closing paragraph (2 sentences): the workflow only chains four templates and one config flag — once this cycle is familiar, the other templates (`Subject Hub`, `Mark Reviewed`, `Update Note Status`, etc.) are documented in the Template Reference section that follows.
